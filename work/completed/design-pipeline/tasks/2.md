# Task 2: Create font-pairing.md reference

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:skill-master` — [skills/skill-master/SKILL.md](~/.claude/skills/skill-master/SKILL.md)

## Description

Создать справочный файл `font-pairing.md` с правилами подбора шрифтовых пар для веб-проектов. Файл будет использоваться скиллом `design-system-init` на этапе интервью по типографике — когда агент предлагает пользователю пары heading+body шрифтов из Google Fonts.

Файл должен содержать практические знания о типографике: принципы контраста и конкорданса при подборе пар, иерархию весов, типографические шкалы (ratios), а также готовые классические комбинации с обоснованием выбора. Это новый reference — аналога в проекте нет, создаётся с нуля на основе устоявшихся типографических знаний.

Согласно Decision 10, файл размещается в `skills/design-system-init/references/`, а не в `shared/design-references/`, потому что только design-system-init использует его при подборе шрифтов. Другие скиллы (design-generate, design-review) работают с уже выбранными шрифтами из tokens.json.

## What to do

1. Создать директорию `skills/design-system-init/references/` (если не существует).

2. Создать файл `skills/design-system-init/references/font-pairing.md` со следующими разделами:

   - **Pairing Principles** — два основных подхода: contrast pairing (heading sans-serif + body serif или наоборот) и concordance pairing (оба из одного семейства/класса). Когда какой использовать, преимущества и ограничения каждого.

   - **Weight Hierarchy** — правила распределения весов между heading и body: heading bold/semibold (600-700), body regular/medium (400-500). Как вес создаёт визуальную иерархию. Минимум 2 уровня контраста (200+ единиц разницы).

   - **Typographic Scale Ratios** — стандартные шкалы размеров: minor third (1.200), major third (1.250), perfect fourth (1.333), golden ratio (1.618). Для каждого — когда применять и какие размеры получаются при base 16px.

   - **Classic Combinations** — 8-12 проверенных пар Google Fonts с обоснованием. Для каждой пары: heading font + body font, тип пейринга (contrast/concordance), настроение (formal, modern, friendly, etc.), рекомендуемые веса, пример use-case. Покрыть основные настроения: деловой, современный, дружелюбный, элегантный, технический, креативный.

   - **Alignment with tokens.json** — как выбранные шрифты маппятся на `typography.families.heading` и `typography.families.body` в tokens.json. Формат записи: `"Font Name, fallback-stack"`.

3. Убедиться, что все рекомендованные шрифты доступны в Google Fonts (бесплатные, без лицензионных ограничений).

4. Убедиться, что файл согласован со схемой typography в design-tokens.md (секция typography: families, sizes, weights, lineHeights).

**Edge cases:**
- Шрифты могут быть удалены из Google Fonts — рекомендовать только давно существующие, стабильные шрифты (Inter, Roboto, Open Sans, Lora, Playfair Display, Merriweather, etc.)

**Implementation hints:**
- Структура файла должна быть пригодна для использования агентом во время интервью: агент читает font-pairing.md и на основе выбранного пользователем настроения предлагает 2-3 подходящие пары
- Каждая комбинация должна быть самодостаточной: прочитав её, агент может сразу записать в tokens.json без дополнительных поисков
- Типографические шкалы (ratios) должны давать конкретные значения размеров при base 16px, чтобы агент мог заполнить `typography.sizes` в tokens.json
- Размер файла — около 180 строк. Это reference-документ, не SKILL.md, поэтому ограничение в 500 строк не применяется, но файл должен быть компактным и практичным
