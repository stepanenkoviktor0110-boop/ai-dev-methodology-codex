# Task 5: Create design-review skill

## Required Skills

- `/skill:skill-master` — [skills/skill-master/SKILL.md](../../../skills/skill-master/SKILL.md)

## Description

Создаём informational skill `design-review` для lightweight ревью UI-кода на соответствие дизайн-токенам проекта.

Скилл вызывается из do-task (Task 7 добавит интеграцию) после коммита кода, когда в проекте есть `.design-system/tokens.json` и задача затронула UI-файлы (.tsx/.vue/.html/.css/.scss). Задача design-review — прочитать только tokens.json (не всю дизайн-систему), просканировать изменённые файлы на hardcoded значения и выдать 2-3 конкретных рекомендации по замене на токены.

Ключевое ограничение: минимальный overhead по токенам (~500-1000 tokens). Скилл не запускает полный пайплайн, не генерирует макеты, не проводит интервью, не создаёт компоненты.

Тип скилла: informational (Decision 4) — нет строгой последовательности фаз, секции организованы по логике, с decision frameworks.

## What to do

1. Создать директорию `skills/design-review/`
2. Создать `skills/design-review/SKILL.md` как informational skill (следуя skill-master guidelines для informational типа)
3. SKILL.md frontmatter: name, description, trigger phrases ("design review", "проверь стили", "проверь токены")
4. Секции SKILL.md:
   - **When to activate**: conditions — `.design-system/tokens.json` exists AND task changed UI files (.tsx/.vue/.html/.css/.scss). Skip silently if either condition fails
   - **What to read**: only `tokens.json` from project's `.design-system/` — not full DS, not components, not references (token economy)
   - **What to check**: hardcoded colors (hex values not from tokens), hardcoded spacing (px values not matching DS spacing scale), hardcoded font families/sizes not from typography tokens, non-DS CSS custom properties
   - **How to report**: 2-3 concrete recommendations in format "found X -> use Y" (e.g., "color #3B82F6 -> use --color-primary-500", "gap 8px -> DS minimum --space-4 (16px)")
   - **When clean**: if 0 findings — report "styles match DS" and stop, no iterations
   - **Scope guard**: explicitly state what this skill does NOT do (no full pipeline, no mockups, no interview, no new components)
5. Добавить ссылки на shared references: design-tokens.md (token schema), color-principles.md (color matching knowledge)

**Dependencies:**

**Token schema для review (из design-tokens.md):**
- colors: primary/secondary (50-900 scale), neutral (0-1000), semantic (success/warning/error/info), background, text
- typography: families (heading/body/mono), sizes (xs-4xl), weights, lineHeights
- spacing: scale 1-16 (4px-64px)
- radii, shadows, breakpoints

**Design Review flow (из tech-spec):**
1. Check: `.design-system/tokens.json` exists AND task changed UI files (.tsx/.vue/.html/.css/.scss)
2. Read only `tokens.json` (not full DS — token economy)
3. Scan changed files for hardcoded values, non-DS colors/spacing/fonts
4. Output 2-3 concrete recommendations: "color #3B82F6 -> use --color-primary-500", "gap 8px -> DS minimum --space-4 (16px)"
5. Apply fixes -> commit

**Informational skill structure (Decision 4):**
- Нет строгих фаз с чекпоинтами (в отличие от procedural)
- Секции по логике: When to activate, What to read, What to check, How to report
- Decision frameworks: YES if / NO if для каждого условия

**Edge cases (из tech-spec scenario tests):**
- UI files with hardcoded colors — gives concrete token recommendations
- Files already using DS tokens — reports "styles match DS"
- No `.design-system/tokens.json` — skips silently (does not error)
- No UI files changed — skips silently

**Scope guard (Decision 5, user-spec "Чего НЕ происходит"):**
- Не запускается полный пайплайн
- Не генерируются макеты
- Не проводится интервью
- Не запускается ретроспектива
- Не создаются новые компоненты

**Implementation hints:**
- Ориентируйся на informational-skills.md из skill-master references для структуры
- Token budget ~500-1000 tokens overhead — скилл должен быть компактным
- Ссылки на shared references используют относительные пути от skills/design-review/
- Конкретные примеры рекомендаций из tech-spec: "color #3B82F6 -> use --color-primary-500", "gap 8px -> DS minimum --space-4 (16px)"
