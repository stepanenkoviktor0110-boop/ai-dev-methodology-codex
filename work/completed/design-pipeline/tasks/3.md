# Task 3: Create design-system-init skill

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:skill-master` — [skills/skill-master/SKILL.md](../../../skills/skill-master/SKILL.md)

## Description

Создать процедурный скилл `design-system-init` — основной инструмент для создания дизайн-системы через сканирование проекта и интервью с пользователем. Скилл мигрирует Phase 1-2 из like-figma, обогащая их: подбор палитры через эмоциональную карту цветов (color-psychology.md), подбор шрифтовых пар (font-pairing.md), опции golden ratio/Fibonacci для spacing (из grid-techniques).

Результат работы скилла: `.design-system/tokens.json` + HTML-файлы компонентов в `.design-system/components/`, каждый из которых открывается автономно в браузере.

Скилл вызывается напрямую через `/design-system-init` и является каноническим (Decision 3: без shim). Интервью встроено в SKILL.md (Decision 2: без interview template file).

## What to do

1. Создать директорию `skills/design-system-init/` (mkdir -p — безусловно, идемпотентно)
2. Написать `skills/design-system-init/SKILL.md` — процедурный скилл со следующими фазами:
   - **Phase 0: Project Readiness** — проверить что проект веб (есть HTML/CSS/JS/TS/React/Vue). Не-веб проект -- отказ с объяснением. Если `.design-system/` уже существует -- спросить: обновить / пересоздать / отменить
   - **Phase 1: Project Scan** — сканирование проекта на существующие стили. Приоритет источников: Tailwind config > CSS vars > SCSS > hardcoded значения. Представить находки пользователю
   - **Phase 2: Interview** — пошаговое интервью по темам (одна тема за раз):
     - Контекст проекта и настроение (для кого, какой характер) -- с использованием эмоциональной карты из color-psychology.md
     - Палитра (primary, secondary, neutrals, semantic) -- предложение на основе настроения, принципы из color-principles.md
     - Типографика -- подбор шрифтовых пар через font-pairing.md (Google Fonts, heading+body)
     - Spacing -- предложить варианты: golden ratio (1.618), Fibonacci, стандартная шкала (4/8/12/16/24/32/48/64)
     - Radii, shadows, breakpoints
     - Компоненты -- какие нужны (предложить на основе скана)
   - **Phase 3: Build** — генерация `tokens.json` по схеме из design-tokens.md + HTML-файлы компонентов (самодостаточные, с inline CSS через CSS custom properties)
   - **Phase 4: Verify** — валидация JSON, проверка CSS custom properties usage в компонентах, contrast ratio >= 4.5:1
3. YAML frontmatter: name: design-system-init, description с trigger phrases на русском и английском
4. Ссылки на references в тексте SKILL.md по Pattern A (action-embedded) -- каждая ссылка встроена в шаг, где используется
5. Убедиться что SKILL.md < 500 строк. При необходимости вынести детали в references
6. Валидация файловых имён компонентов: `/^[a-z0-9-]+$/` (Decision 9)
7. Запустить skill-checker для валидации

- Task 2 (font-pairing.md) — `skills/design-system-init/references/font-pairing.md` должен существовать

**Edge cases:**
- `.design-system/` уже существует в проекте пользователя -- скилл должен спросить: обновить / пересоздать / отменить
- Конфликтующие значения стилей (Tailwind config говорит одно, CSS vars другое) -- показать конфликт, спросить приоритет
- Проект без CSS/стилей вообще -- предложить дефолтные значения, интервью с нуля
- Не-веб проект -- отказ с объяснением (MVP: только веб)
- Пользователь хочет тёмную тему без dark-токенов -- спросить, не принимать решение самостоятельно

**Implementation hints:**
- Миграция из like-figma Phase 1 (Project Scan) + Phase 2 (Build Design System) -- взять структуру, но обогатить interview темами mood/psychology
- Interview embedded directly in SKILL.md (Decision 2) -- не создавать отдельный interview template
- Ссылки на references по Pattern A (action-embedded): "propose palette based on emotional map from [color-psychology.md](...)", "select font pairs following rules from [font-pairing.md](...)"
- Для spacing предложить 3 опции: golden ratio (1.618x scale), Fibonacci (1,1,2,3,5,8,13,21 -> 4,4,8,12,20,32,52,84 px), стандартная шкала (4/8/12/16/24/32/48/64)
- Tokens.json schema -- точно по design-tokens.md (colors, typography, spacing, radii, shadows, breakpoints)
- Компоненты: self-contained HTML + inline CSS с CSS custom properties. Каждый файл открывается в браузере автономно
- Валидация имён файлов компонентов: `/^[a-z0-9-]+$/` перед созданием (Decision 9 -- предотвращение path traversal)
- Contrast ratio check: текст/фон пары из tokens.json, все >= 4.5:1 (WCAG AA)
- Trigger phrases для description: "создай дизайн-систему", "инициализируй дизайн", "design system init", "create design system", "настрой дизайн"
