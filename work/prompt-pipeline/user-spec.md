# User Spec: prompt-pipeline

## Что делаем

n8n workflow для пошаговой разработки проектов через copy-paste в бесплатные LLM (ChatGPT, DeepSeek, Qwen). Workflow ведёт пользователя по шагам методологии: от описания идеи до готового набора файлов проекта. На каждом шаге генерируется промт, пользователь копирует его в LLM, вставляет ответ обратно, n8n парсит результат и создаёт файлы.

## Зачем

Когда лимиты Claude Code заканчиваются, а проект делать надо — нет способа продолжить работу по методологии. Prompt Pipeline даёт альтернативу: тот же пайплайн (спека → декомпозиция → генерация кода), но через любую бесплатную LLM без API-ключей.

## Как должно работать

### Основной сценарий

1. Пользователь открывает n8n workflow и запускает его (форма)
2. Форма "Новый проект": выбор шаблона из списка (~13 штук) + описание проекта в свободной форме + путь для файлов (настраивается, работает и на VPS и локально)
3. n8n генерирует **промт для спеки** (описание + контекст шаблона) и показывает его
4. Пользователь копирует промт в LLM → получает спеку → вставляет ответ в n8n
5. n8n генерирует **промт для валидации спеки** ("проверь на пробелы и противоречия") и показывает его
6. Пользователь копирует → получает замечания → вставляет → n8n обновляет спеку
7. n8n генерирует **промт для декомпозиции** (разбить на 5-8 файлов MVP, остальное в бэклог)
8. Пользователь копирует → получает список файлов → вставляет
9. Для каждого файла (шаги 10-12 повторяются):
10. n8n генерирует **промт для файла N** с контекстом: спека + содержимое уже готовых файлов 1..N-1
11. Пользователь копирует → получает код → вставляет
12. (Опционально) Если галочка "Ревью" включена — n8n генерирует промт для code review файла, пользователь копирует → вставляет замечания → n8n показывает промт для исправлений
13. Все файлы готовы → n8n создаёт файлы по указанному пути + предлагает скачать архив
14. Workflow завершён, лог сессии сохранён

### Шаблоны проектов (~13 штук)

1. Лендинг (HTML/CSS/JS)
2. Многостраничный сайт (HTML/CSS/JS)
3. Telegram-бот (Python)
4. Telegram-бот (Node.js)
5. Вебап (Next.js)
6. Вебап (React SPA)
7. API-сервер (FastAPI)
8. API-сервер (Express)
9. CLI-утилита (Python)
10. CLI-утилита (Node.js)
11. n8n: webhook + обработка данных
12. n8n: интеграция с API (Telegram, email, etc.)
13. n8n: scheduled workflow (крон-задачи)

Каждый шаблон определяет: типичную структуру файлов (5-8 штук), стек, ограничения размера, контекст для промтов.

### Формат ответа LLM

В каждом промте есть инструкция для LLM — оборачивать файлы маркерами:

```
=== FILE: path/to/file.ext ===
содержимое файла
=== END FILE ===
```

n8n парсит по маркерам. Если маркеров нет — весь ответ считается одним файлом.

### Логирование

Параллельно с основным flow n8n пишет JSON-лог сессии:
- Выбранный шаблон и описание проекта
- Каждый сгенерированный промт
- Каждый ответ LLM (вставленный пользователем)
- Количество попыток на каждый шаг (повторы)
- Таймстемпы всех шагов
- Итоговая структура файлов

Лог используется для ретроспективы: скармливается в LLM с промтом анализа.

## Критерии приёмки

- [ ] Workflow запускается в n8n, показывает форму выбора шаблона
- [ ] Все 13 шаблонов доступны для выбора
- [ ] Путь для файлов настраивается (VPS и локальный)
- [ ] На каждом шаге генерируется промт с правильным контекстом
- [ ] Промт для файла N содержит контекст файлов 1..N-1
- [ ] Ответ LLM парсится по маркерам `=== FILE ===` / `=== END FILE ===`
- [ ] Файлы создаются по указанному пути с правильной структурой
- [ ] Доступна кнопка "Скачать архив" по завершении
- [ ] Кнопка "Повторить шаг" работает при плохом ответе LLM
- [ ] Опциональный code review на каждом шаге (галочка)
- [ ] Валидация спеки работает (отдельный промт)
- [ ] Декомпозиция ограничивает MVP до 5-8 файлов, остальное в бэклог
- [ ] JSON-лог сессии сохраняется автоматически
- [ ] Промты хранятся в отдельных .md файлах (не внутри n8n нод)

## Ограничения

- Только бесплатные LLM через веб-сайт (chatgpt.com, chat.deepseek.com, Qwen). Без API-ключей.
- Один проект одновременно.
- MVP проекта: максимум 5-8 файлов. Ограничение задаётся шаблоном.
- Размер файлов ограничен шаблоном — все файлы должны помещаться в контекст LLM при подстановке.
- Россия: всё что требует заблокированных API — не используется.

## Риски

- **LLM не следует формату маркеров.** Митигация: кнопка "Повторить шаг" с тем же промтом. В промте — чёткая инструкция с примером формата.
- **Ответ LLM обрезан (контекст переполнен).** Митигация: шаблоны проектируются так, чтобы контекст (спека + готовые файлы) не превышал разумного размера. Файлы маленькие по определению шаблона.
- **Качество кода от бесплатных LLM ниже Claude Code.** Митигация: опциональный шаг code review, валидация спеки. Принимаем как осознанный компромисс.

## Технические решения

- Платформа — n8n workflow на VPS (уже установлен).
- Промты хранятся в отдельных .md файлах на диске, а не внутри n8n нод — проще редактировать.
- Copy-paste вместо API — осознанный выбор: бесплатно, без блокировок, работает с любой LLM.
- Один workflow на весь пайплайн (не набор отдельных workflows).
- Файлы проекта создаются на VPS + опция скачать архив.

## Тестирование

**Unit-тесты:** парсер ответов LLM (маркеры `=== FILE ===`). Это единственный критичный компонент — если парсер сломается, файлы не создадутся.

**Интеграционные тесты:** не делаем — n8n workflow тестируется вручную прогоном.

**E2E тесты:** не делаем — интерфейс это формы n8n, тестировать их автоматически нецелесообразно.

## Как проверить

### Агент проверяет

| Шаг | Инструмент | Ожидаемый результат |
|-----|-----------|-------------------|
| 1. Импортировать workflow в n8n | bash (n8n CLI или API) | Workflow появился в списке |
| 2. Запустить парсер на тестовом ответе LLM | bash (node script) | Файлы распарсены корректно |
| 3. Проверить что .md файлы промтов читаются | bash (cat) | Все промты на месте |

### Пользователь проверяет

- Пройти полный цикл: выбрать шаблон "Лендинг" → сгенерировать промты → скопировать в LLM → вставить ответы → убедиться что файлы созданы и лог записан
- Проверить кнопку "Повторить шаг" — вставить мусор вместо ответа, нажать повторить
- Скачать архив и убедиться что структура файлов корректна
