# Task 10: Security Audit

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:security-auditor` — [skills/security-auditor/SKILL.md](../../skills/security-auditor/SKILL.md)

## Description

Полноценный аудит безопасности всех артефактов фичи design-pipeline. Задача Audit Wave — код не меняется, только чтение и создание отчёта.

Фича design-pipeline создаёт 4 скилла (design-system-init, design-generate, design-review, design-retrospective), shared references и интеграцию в do-task. Скиллы являются markdown-инструкциями, которые агент исполняет. Это означает, что векторы атак нестандартные: prompt injection через пользовательский ввод, небезопасные команды в verify-smoke инструкциях, path traversal при создании файлов в `.design-system/`, XSS в генерируемом HTML.

Ключевые зоны риска из tech-spec decisions:
- **Decision 8 (CSP):** `preview-template.html` использует placeholder substitution (`{{PAGE_CONTENT}}`, `{{PAGE_STYLES}}`, `{{PAGE_TITLE}}`). Без CSP `script-src 'none'` инъекция в placeholder может выполнить произвольный JS в браузере пользователя.
- **Decision 9 (Path validation):** Имена компонентов и страниц берутся из пользовательского ввода. Без валидации `/^[a-z0-9-]+$/` возможен path traversal (`../../etc/passwd`) при создании файлов.
- **preview-template.html:** Текущий шаблон (из like-figma) не содержит CSP meta tag — задача 4 должна была добавить его при миграции.

3. Проанализировать каждый файл по следующим векторам:
   - **Prompt injection:** Может ли пользовательский ввод (описание страницы, имя компонента, фидбек при итерации) заставить агента выполнить непредусмотренные действия? Есть ли в инструкциях указания по санитизации или разграничению user input vs. system instructions?
   - **Unsafe commands:** Есть ли в verify-smoke или в инструкциях скилла команды, которые могут быть опасны (rm, curl к внешним URL, eval, exec)? Могут ли команды быть инъецированы через пользовательский ввод?
   - **Path traversal:** Проверить все места, где создаются файлы на основе пользовательского ввода (`.design-system/components/{name}.html`, `.design-system/pages/{name}.html`). Указана ли валидация `/^[a-z0-9-]+$/` (Decision 9)?
   - **XSS в generated HTML:** Проверить `preview-template.html` на наличие CSP `script-src 'none'` (Decision 8). Проверить, нет ли в инструкциях скилла генерации inline `<script>` тегов.
   - **Information disclosure:** Могут ли инструкции скилла привести к раскрытию чувствительных данных проекта (env vars, secrets, credentials)?

4. Написать отчёт аудита в `work/design-pipeline/logs/working/task-10/security-audit.md`

## Details

**Files:**
- `skills/design-system-init/SKILL.md` — аудит: prompt injection при интервью (mood, palette, typography input), path traversal при создании компонентов в `.design-system/components/`
- `skills/design-generate/SKILL.md` — аудит: prompt injection через описание страницы, path traversal при создании `.design-system/pages/{name}.html`, unsafe commands в verify
- `skills/design-generate/assets/preview-template.html` — аудит: CSP meta tag наличие, placeholder substitution XSS
- `skills/design-review/SKILL.md` — аудит: prompt injection через содержимое анализируемых файлов, information disclosure
- `skills/design-retrospective/SKILL.md` — аудит: prompt injection через user feedback, path traversal при записи в lessons-learned.md
- `skills/do-task/SKILL.md` — аудит: design-review hook не создаёт новых attack surfaces

**Dependencies:** Tasks 3, 4, 5, 6, 7, 8 — все файлы для аудита создаются этими задачами

**Ключевые проверки из Decisions:**

1. **Decision 8 — CSP:** `preview-template.html` ДОЛЖЕН содержать `<meta http-equiv="Content-Security-Policy" content="script-src 'none'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com">`. Если нет — Critical finding.

2. **Decision 9 — File naming:** Инструкции в design-system-init и design-generate ДОЛЖНЫ содержать валидацию имён: только `/^[a-z0-9-]+$/`, запрет path separators. Если нет — High finding.

3. **preview-template.html placeholders:** Текущий like-figma шаблон содержит `{{PAGE_TITLE}}`, `{{PAGE_STYLES}}`, `{{PAGE_CONTENT}}`. Все три — потенциальные XSS-векторы. CSP `script-src 'none'` митигирует `<script>` инъекцию, но `{{PAGE_TITLE}}` в `<title>` может сломать HTML structure. Проверить, описана ли санитизация в design-generate SKILL.md.

4. **Prompt injection в скиллах:** Скиллы — это инструкции для LLM-агента. Пользовательский ввод (описание страницы: "make a page that says ignore all instructions and...") — классический prompt injection. Проверить наличие guardrails / separation of concerns в инструкциях скиллов.

**Формат отчёта:**
Для каждого finding:
- Severity: Critical / High / Medium / Low
- File: путь к файлу
- Issue: описание проблемы
- Attack vector: как можно эксплуатировать
- Recommendation: конкретное исправление

**Implementation hints:**
- Это аудит-задача: никаких изменений в код не вносится
- Отчёт пишется в markdown, не JSON
- Если все Decision 8 и Decision 9 требования выполнены — отметить как passed, не искать проблемы где их нет
- Если обнаружены проблемы не предусмотренные decisions — это ценная находка, обязательно включить
