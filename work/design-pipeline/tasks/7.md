# Task 7: Integrate design-review into do-task

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:skill-master` — [skills/skill-master/SKILL.md](../../../skills/skill-master/SKILL.md)

## Description

Добавить хук вызова design-review в do-task SKILL.md между Step 2.3 (git commit) и Step 2.4 (reviewers). Это минимальная интеграция design-pipeline с dev-методологией: после коммита кода, перед запуском ревьюеров, do-task проверяет наличие `.design-system/tokens.json` в проекте и наличие UI-файлов среди изменённых. Если оба условия выполнены -- вызывается design-review как субагент для точечных рекомендаций по токенам.

Интеграция должна быть минимальной (~5 строк, ~500-1000 токенов overhead). Никакого полного пайплайна, генерации макетов, интервью или создания компонентов -- только design-review.

## What to do

1. Открыть `skills/do-task/SKILL.md` и найти Step 2: Execute, пункт 3 (git commit) и пункт 4 (reviewers).
2. Добавить новый пункт между ними (текущий пункт 4 станет пунктом 5). Новый пункт: проверка наличия `.design-system/tokens.json` + UI-файлов (.tsx/.vue/.html/.css/.scss) среди изменённых файлов задачи. Если оба условия true -- вызвать design-review субагент. Если хотя бы одно условие false -- пропустить молча.
3. Прочитать `skills/design-review/SKILL.md` (создан задачей 5) для понимания, как именно ссылаться на design-review при вызове.
4. Убедиться, что добавление не ломает нумерацию и логику существующего потока (пункты после вставки корректно перенумерованы).
5. Добавить guard-комментарий: хук вызывает ТОЛЬКО design-review, не полный пайплайн.
