# Task 4: Create design-generate skill

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:skill-master` — [skills/skill-master/SKILL.md](../../../skills/skill-master/SKILL.md)

## Description

Создать процедурный скилл `design-generate` для генерации HTML+SVG макетов из текстовых описаний пользователя. Скилл мигрирует Phase 3 из like-figma с обогащением: вместо 5 базовых лейаутов доступно 15 паттернов компоновки (5 basic + 10 advanced grids из grid-techniques.md). Скилл собирает страницы из компонентов дизайн-системы, применяет токены, использует preview-template.html для интерактивного HTML-превью, генерирует статический SVG (1440x900 desktop / 390x844 mobile). Поддерживает итерацию через скриншоты пользователя и генерирует continuation prompt при исчерпании контекста.

Этот скилл является одним из 4 MVP-скиллов design-pipeline и зависит от наличия дизайн-системы (`.design-system/tokens.json`), созданной через `/design-system-init`.

## What to do

1. Создать директорию `skills/design-generate/` со структурой: `SKILL.md`, `references/`, `assets/`

2. Скопировать и адаптировать `references/component-patterns.md` из like-figma (`skills/like-figma/references/component-patterns.md`). Содержимое переносится as-is — это каталог компонентов (Button, Input, Card, Badge, Alert, Avatar, Nav, Table) с BEM-naming, вариантами, слотами, CSS-паттернами на токенах

3. Скопировать и обогатить `references/generation-guide.md` из like-figma (`skills/like-figma/references/generation-guide.md`). Исходный файл содержит 5 базовых лейаутов (Single Column, Sidebar+Content, Grid, Hero+Sections, Split Screen), процесс сборки страниц, responsive behavior, placeholder content, HTML/SVG generation. Обогащение: добавить секцию "Advanced Grid Patterns" со ссылкой на grid-techniques.md, расширить секцию Layout Patterns таблицей выбора "описание запроса → рекомендуемый паттерн" для всех 15 лейаутов, добавить guidance по выбору между basic и advanced

4. Скопировать `references/grid-techniques.md` из `work/design-pipeline/references/grid-techniques.md`. Содержит 10 нестандартных сеток: золотое сечение, Fibonacci, broken grid, Van de Graaf, ratio grid, optical margin, diagonal layout, typographic grid, bento с нарушенной симметрией, Swiss grid с цветовыми полями. Каждая с CSS-примерами

5. Скопировать и доработать `assets/preview-template.html` из like-figma (`skills/like-figma/assets/preview-template.html`). Добавить CSP meta tag (Decision 8): `<meta http-equiv="Content-Security-Policy" content="script-src 'none'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com">`. Текущий шаблон содержит CSS reset, placeholder-токены в :root, placeholders `{{PAGE_TITLE}}`, `{{PAGE_STYLES}}`, `{{PAGE_CONTENT}}`

6. Написать `SKILL.md` — процедурный скилл со следующими фазами:
   - **Phase 0: Readiness Check** — проверить наличие `.design-system/tokens.json`, валидность JSON. Если нет — отказ с предложением `/design-system-init`. Если JSON невалидный — ошибка с предложением пересоздать
   - **Phase 1: Parse Request** — разобрать описание пользователя: тип страницы, нужные компоненты, структура лейаута, контент (реальный или placeholder). При неоднозначности — предложить конкретную интерпретацию и подтвердить
   - **Phase 2: Select Layout** — выбрать лейаут из 15 паттернов, применяя guidance из generation-guide.md. Для basic layouts — встроенные CSS. Для advanced — применить технику из grid-techniques.md
   - **Phase 3: Assemble & Generate** — собрать страницу из компонентов (component-patterns.md), применить токены. Сгенерировать HTML (используя preview-template.html) + SVG (статика). Валидация имён файлов: только `[a-z0-9-]+` (Decision 9)
   - **Phase 4: Present & Iterate** — показать результат, предложить открыть HTML в браузере. Пользователь присылает скриншот с фидбеком — итерировать. Если контекст исчерпывается — остановиться и сгенерировать continuation prompt
   - **Checkpoints** после каждой фазы
