## Description

Full-feature code quality audit for the design-pipeline feature. This is an Audit Wave task -- no code changes, only reading and producing an audit report.

The design-pipeline feature creates 4 new design skills (design-system-init, design-generate, design-review, design-retrospective), shared design references, font-pairing reference, preview template, and a do-task integration hook. This audit reviews all created/modified artifacts holistically for cross-skill consistency, structural compliance with skill-master patterns, reference link validity, naming conventions, and tone consistency.

The audit catches issues that per-task skill-checker reviews may miss: cross-skill inconsistencies, broken inter-skill references, divergent naming conventions, and holistic quality problems.

## What to do

1. Read all 4 SKILL.md files: design-system-init, design-generate, design-review, design-retrospective
2. Read all reference files: shared/design-references/ (design-tokens.md, color-psychology.md, color-principles.md, library-catalog.md), skills/design-system-init/references/font-pairing.md, skills/design-generate/references/ (component-patterns.md, generation-guide.md, grid-techniques.md)
3. Read the preview template: skills/design-generate/assets/preview-template.html
4. Read the updated do-task/SKILL.md to verify the design-review integration hook
5. Read skill-master/SKILL.md and its references (procedural-skills.md, informational-skills.md) as the standard to audit against
6. Audit each SKILL.md for structural compliance with skill-master: frontmatter format, skill type adherence (procedural vs informational), phase structure, checkpoints, line count (<500 lines)
7. Audit cross-skill consistency: naming conventions for tokens/components/files, consistent terminology across all 4 skills, consistent tone and writing style
8. Validate all reference links: every path referenced in SKILL.md files must point to an existing file. Check both relative and absolute paths
9. Verify shared references are not duplicated inside individual skill directories
10. Check do-task integration: hook is minimal (~5 lines), placed correctly (between Step 2.3 and 2.4), guard conditions present
11. Write audit report to `logs/working/task-9/code-audit-report.json` with findings categorized by severity (critical/major/minor)

## Details

**Nature of task:** This is a read-only audit. No files are modified. The only output is an audit report JSON file.

**Files to read (all created by tasks 1-8):**
- `skills/design-system-init/SKILL.md` -- procedural skill, must follow skill-master Phase structure with checkpoints
- `skills/design-generate/SKILL.md` -- procedural skill, must follow skill-master Phase structure with checkpoints
- `skills/design-review/SKILL.md` -- informational skill, must follow skill-master section/decision-framework structure
- `skills/design-retrospective/SKILL.md` -- procedural skill, must follow skill-master Phase structure with checkpoints
- `shared/design-references/` -- 4 shared reference files (design-tokens.md, color-psychology.md, color-principles.md, library-catalog.md)
- `skills/design-system-init/references/font-pairing.md` -- skill-specific reference
- `skills/design-generate/references/` -- 3 reference files (component-patterns.md, generation-guide.md, grid-techniques.md)
- `skills/design-generate/assets/preview-template.html` -- HTML template with CSP
- `skills/do-task/SKILL.md` -- verify ~5 lines added between Step 2.3 and 2.4

**Dependencies:** Tasks 3-8 must be completed (all skills created, do-task integrated, like-figma deleted).

**Audit dimensions (adapted from code-reviewing for skill/reference content):**
1. **Structural compliance** -- each SKILL.md follows skill-master patterns (correct skill type, frontmatter, phases/sections, checkpoints, <500 lines)
2. **Cross-skill consistency** -- naming conventions (token names, component names, file patterns), terminology, writing style, tone
3. **Reference link validity** -- every `[text](path)` in SKILL.md resolves to an existing file
4. **Content quality** -- references contain substantive, actionable content (not boilerplate)
5. **No duplication** -- shared references are not copied into individual skill directories
6. **Integration correctness** -- do-task hook is minimal, correctly placed, has guard conditions
7. **Security** -- preview-template.html has CSP `script-src 'none'`, file name validation pattern present in relevant skills
8. **Tech-spec alignment** -- implementation matches decisions from tech-spec (Decision 1-11)

**Report format:** JSON with structure:
```json
{
  "task": 9,
  "type": "code-audit",
  "date": "YYYY-MM-DD",
  "summary": "1-3 sentence summary",
  "findings": [
    {
      "severity": "critical|major|minor",
      "file": "path/to/file",
      "issue": "description",
      "recommendation": "what to fix"
    }
  ],
  "statistics": {
    "files_reviewed": N,
    "findings_critical": N,
    "findings_major": N,
    "findings_minor": N
  }
}
```

**Edge cases:**
- If like-figma directory still exists (task 8 incomplete) -- flag as critical finding
- If any SKILL.md exceeds 500 lines -- flag as major finding
- If reference links use backslashes (Windows paths) -- flag as minor finding
