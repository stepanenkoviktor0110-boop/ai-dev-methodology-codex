# Quick Reference — Feature Execution

1. Верифицируй результат в реальной среде перед объявлением "готово" (Seen: 4)
2. Серверные операции — проверяй preconditions: порты, push state, SSH, pkill в deploy script перед restart (Seen: 2)
3. Субагент не завершает задачу → lead выполняет напрямую (Seen: 2)
4. Спорные решения ДО генерации: выписать, утвердить, потом генерировать (Seen: 2)
5. Любое изменение — проверяй радиус поражения: sibling-группы, CORS/CSRF при смене домена, audit wave для hotfix (Seen: 2)
6. Пользовательские инструкции — уточняй scope: "не нужен" → UI или вся функциональность? (Seen: 2)
7. Не подтверждай без проверки реальности: "это работает?" → grep в коде до ответа (Seen: 2)
8. depends_on=[N] в той же wave → проверить wave(dep) < wave(task) (Seen: 2)
9. Баг после деплоя → проверить upstream-инфраструктуру до повторного анализа кода (Seen: 1)
10. Внешний агент → проверить полный git diff, не только целевые файлы (Seen: 1)
