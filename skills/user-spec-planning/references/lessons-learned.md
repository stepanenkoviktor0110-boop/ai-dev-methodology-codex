# Lessons Learned: user-spec-planning

### 2026-03-26 design-pipeline-v1: AC testability требует 2 раунда валидации

**Problem:** User-spec прошёл 2 раунда валидации — после первого раунда осталось 2 critical AC с проблемами тестируемости.
**Cause:** Acceptance criteria описывались на уровне поведения агента ("агент предлагает из предпочтений"), а не на уровне верифицируемых артефактов (файл/секция/шаг).
**Solution:** AC переписаны в формат "SKILL.md содержит шаг X", "файл Y содержит секцию Z" — проверяемые grep/read.
**Rule:** При написании AC для markdown-only фич (скиллы, reference-файлы) формулировать критерии через наличие конкретных артефактов: "файл X содержит Y", "SKILL.md содержит шаг Z в Phase N" — не через поведение агента, которое невозможно автоматически проверить.
